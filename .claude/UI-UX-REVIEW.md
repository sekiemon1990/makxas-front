# makxas-front UI/UX レビュー（2026-05-19）

## 概要

全画面（15画面・モバイル/デスクトップ）を網羅的に確認した結果のレポート。
評価軸は 6 観点：
1. **視覚階層**（情報の重要度が見て分かるか）
2. **一貫性**（コンポーネント・色・余白の統一）
3. **操作性**（クリックターゲット・フィードバック）
4. **認知負荷**（情報整理・命名・空ステート）
5. **レスポンシブ**（モバイル/タブレット/デスクトップ）
6. **アクセシビリティ**（ラベル・コントラスト・キーボード）

優先度: **A=Critical（業務に支障）** → **B=High（毎日のフラストレーション）** → **C=Medium（磨き上げ）** → **D=Low（細部）**

---

## エグゼクティブサマリー

makxas-front はインサイドセールス向けに**機能は豊富**だが、Phase 1〜Phase 3 にわたって追加した機能が**一貫したデザインシステム無しに積み重なっており**、3つのカテゴリで UX 負債が蓄積している:

1. **情報密度の暴走**: ダッシュボード・インボックスに 10+ のカードや 6+ のボタンが密集し、視線の流れが定まらない
2. **設定画面のスケール限界**: タブ 11個が改行表示。タブ方式の限界を超えている
3. **モバイル / 顧客向け / 管理向けの境界が曖昧**: 顧客フォーム `/inquiry` までもがサイドバーつきの管理UI レイアウトで表示されている

総評: **「機能を使いこなせるパワーユーザー1〜2名向け」のUIになっており、新規スタッフ4〜10名で運用するには認知負荷が高い。Information Architecture（IA）の再設計が必要**。

## A: Critical 問題（即対応すべき）

### A1. 設定画面のタブ 11個が改行されて読めない

**現状:**
- 「ブランド管理 / 店舗管理 / LINEアカウント / メールアカウント / 比較サイト / スタッフ管理 / 返信テンプレート / 目標設定 / 営業時間 / AI設定 / フィードバック」
- 1456px の標準デスクトップ幅で**全タブが2行に改行**。「ブランド管/理」のように単語分割発生
- 新しい設定カテゴリを追加するたびに更に窮屈になる構造

**影響:** 設定にアクセスするたびに認知負荷。新規スタッフは「どこに何があるか」を毎回探す。

**改善案（推奨）:**
- **左サイドバー型ナビゲーション**に変更（カテゴリ別グルーピング）
  ```
  ┌─ 組織       ┬─ ブランド管理
  │             ├─ 店舗管理
  │             └─ スタッフ管理
  ├─ チャネル   ┬─ LINEアカウント
  │             ├─ メールアカウント
  │             └─ 比較サイト
  ├─ 業務設定   ┬─ 返信テンプレート
  │             ├─ 目標設定
  │             └─ 営業時間
  └─ AI         ┬─ AI設定
                └─ フィードバック
  ```
- もしくはタブを**カテゴリ別の2段組**に再構成

---

### A2. 公開フォーム `/inquiry` が管理 UI のサイドバー付き

**現状:**
- 顧客向け公開フォーム `/inquiry` を開くと、左にサイドバー（ダッシュボード / インボックス / 設定…）が表示される
- 顧客には「これは何のアプリ?」「私もインボックス見られるの?」と混乱を与える
- アクセシビリティとして「顧客に内部 UI を見せている」状態

**影響:** ブランディング・信頼性低下。社内の情報露出リスク。

**改善案:**
- `/inquiry` は **AppShell を使わない独立レイアウト**にする
- 上部に「買取マクサス」ロゴだけのシンプルなヘッダー
- 完了ページにも問い合わせ後の案内（電話番号など）を表示

---

### A3. インボックス返信欄のボタンが 6個並んでいて混乱

**現状:**
- 「AI下書き / AIに質問 / テンプレート / AI原案を確認 / 画像 / 送信」が横並び
- どれが返信用の主アクションか不明瞭、送信ボタンが右端で目立たない
- AI関連のボタンだけで 3つ（AI下書き / AIに質問 / AI原案を確認）あり、機能が重複しているように見える

**影響:** スタッフが返信フローで毎回迷う。応答時間に影響。

**改善案:**
- **AI関連ボタンを1つの「AI ▼」ドロップダウンに集約**
  - 下書きを生成 / 質問する / 原案を再確認
- **メインアクション「送信」を右端の主要ボタン（プライマリーカラー）として強調**
- セカンダリ操作（テンプレート・画像）は左側にアイコンボタンでコンパクトに

---

### A4. モバイル下部ナビバーがアイコンのみでラベル無し

**現状:**
- モバイル表示で下部に 10個のアイコンが並ぶが、**全部ラベル無し**
- ダッシュボード・インボックス・リード・商品・アポ・シフト・AI・設定・管理・AI学習...と並ぶが、どれがどれか触ってみないと分からない
- アクセシビリティ（スクリーンリーダー対応）も懸念

**影響:** モバイル運用での発見可能性が極端に低い。

**改善案:**
- アイコン下に **小さな日本語ラベル（10px程度）**を追加
- もしくは下部は主要4機能（ダッシュ・インボックス・アポ・AI）だけ、その他はハンバーガーメニューに
- アクセシブルラベル（`aria-label`）の確認

---

### A5. `/admin/ai-chats` でサイドバー2項目が同時アクティブ

**現状:**
- `/admin/ai-chats` を開くと、サイドバーの「AI学習・自動化」と「AI履歴管理」**両方が黒背景でアクティブ表示**
- 現在地が「どっち?」と混乱する

**影響:** ナビゲーションの基本機能が壊れている。

**改善案:**
- アクティブ判定ロジックの修正:
  - 厳密一致（`pathname === href`）か、`startsWith()` + 親優先かを統一
  - 親 `/admin` の子ページに居る時、親ハイライトと子ハイライトの優先順位を明確化

## B: High 問題（毎日のフラストレーション）

### B1. ダッシュボードのカード数が多すぎる（情報の階層なし）

**現状:** 12+ のカードが縦に並ぶ
- サマリーカード4個 / 目標進捗 / チャネル別件数 / 最近の反響 / 日次推移 / アポ数推移 / 5月アポ件数 / 担当者別 / 商品カテゴリ / 追加買取（レバー2）/ 顧客プロファイル / チャネル別アポ率 / 失注タグ / 時間あたり生産性 / スタッフ別生産性

すべて同じ視覚的重み。**「今日、まず何を見るべきか」が伝わらない**。

**改善案:**
- **3階層化**:
  1. **ヒーロー**（最上部）: 今日のアポ + 新着件数 + 追加品確認率（レバー2）を大きく
  2. **要約**: チャネル別 / 担当者別 / 月次目標 進捗
  3. **詳細**: 折りたたみ可能なドリルダウンセクション（生産性レポート・失注分析）
- カードに **「今日見るべきインジケータ」**を追加（前日比の上下矢印）
- セクションヘッダー（「今日」「今月」「分析」）でグルーピング

---

### B2. ステータスバッジの色分けが弱い・凡例なし

**現状:**
- 「新着」「対応中」「アポ取得済」「失注」「クローズ」のバッジ色が薄く、ぱっと見で区別しにくい
- 「対応中」がアンバー、「アポ取得済」が緑だが、新規スタッフは凡例なしには色の意味を学べない

**改善案:**
- 色を**より強くする**（背景・テキスト両方）
- インボックス上部に**ステータス凡例**を小さく表示
- 失注・クローズなど終了系は**グレースケール**で寄り目に

---

### B3. 未返信時間バッジ「⚠️ 170h」が数字だけで意味不明

**現状:**
- インボックス一覧で「⚠️ 170h」と赤字表示
- 「170時間=約7日」を一瞬で理解できない
- 何時間で警告色に変わるかの基準が不明

**改善案:**
- **「7日未返信」「3時間未返信」**のように日本語化
- ホバーで「最後の顧客メッセージから経過した時間」と説明
- 24h以下=黄、72h以下=橙、72h超=赤 のような段階表示

---

### B4. シフト管理画面の空ステートが不親切

**現状:**
- シフト管理を開くと「関 憲人 0h」と1行表示されるだけ、全マスが「+」記号
- 新規スタッフはどこをどう操作すればいいか分からない

**改善案:**
- 中央に**オンボーディング表示**: 「最初にスタッフを追加してください」「シフトはセルをクリックして登録します」
- サンプル表示（薄い色でモックシフトを描き、「これは表示例です」と示す）

---

### B5. 通知ON ボタンの状態が分かりにくい

**現状:**
- 「通知ON」「通知OFF」のラベルだけで、現在のブラウザ通知許可状況が見えない
- 拒否済みの場合の再有効化ガイドがない

**改善案:**
- 状態を3段階で表示: **🔔 ON / 🔕 OFF / ⚠️ ブラウザでブロック中**
- ブロック中時にクリックすると「ブラウザ設定 > 通知 でこのサイトを許可してください」のヘルプを表示

---

### B6. AI関連ボタンの紫色凡例なし

**現状:**
- AI関連 UI が**紫色**で統一されているが、初見では「紫=AI」を学習できない
- AI抽出 / AI生成済み / AI下書き / AI履歴 など同系統だが微妙にUIが違う

**改善案:**
- **AIマーク統一**: スパークルアイコン ✨ + 紫を全AI機能で徹底
- ホバー時に「AI生成のため精度に揺らぎがあります」のヒント

---

### B7. 商品一覧の条件フィルター（N/S/A/B/C/D/J）が解説なし

**現状:**
- 「N」「S」「A」〜「J」の単独文字ボタンが並ぶ
- リユース業界の状態ランクだが、新人は意味が分からない

**改善案:**
- ホバーでツールチップ「N=新品未使用 / S=未使用に近い / ...」
- 略号の隣に小さい解説を併記、もしくは凡例リンク

## C: Medium 問題（磨き上げ）

### C1. 件数表記の不統一
- 「反響一覧 9件」「全 8 件」「3件 件表示」など書式バラバラ
- **「N 件」で統一**

### C2. 余白サイズの不統一
- ダッシュボード: カード間 24px / インボックス: カード間 8px / 設定: パディング 32px
- **8px グリッド**で統一（4 / 8 / 12 / 16 / 24 / 32）

### C3. リード一覧の行クリック領域
- 顧客名をクリックすると詳細に飛ぶが、**行全体がクリック可能ではない**
- 行ホバー時に色変化はするが詳細リンクは名前だけ

### C4. アポ一覧で月跨ぎの予定が見えない
- カレンダー表示は美しいが、6月のアポは「›」ボタンで月送りしないと見えない
- 「リスト表示」と「カレンダー表示」の切替が右上のアイコンで小さい

### C5. タグ表示の一貫性
- インボックス: 角丸4px・薄い背景
- リード詳細: 角丸6px・白背景・枠線
- リード一覧: 円形バッジ
- → **タグデザイントークンを1つに**

### C6. 「目標を編集」のリンクが目立たない
- ダッシュボードの目標進捗カードで、編集導線が薄い灰色テキストリンク
- → ボタン化 or 編集アイコン追加

### C7. 検索バー位置の不統一
- ダッシュボード: なし
- リード一覧: 右上
- 商品一覧: テーブル内
- インボックス: フィルター列の上
- → **画面右上に統一**

### C8. インボックス左カラム「全て / 全チャンネル / 全店舗 / 自分のみ / 通知ON」が密集
- 5個のフィルターチップが横並びで詰まっている
- → グルーピング・改行を整理

### C9. AIアシスタント `/ai` のサジェスト質問が多すぎ
- 4 カテゴリ × 3〜4 質問 = 12+ 質問が初期表示
- → 上位 3〜4 質問だけ表示し、他は「もっと見る」で展開

### C10. リード一覧の「最終接触」が「9日前」と「2026/05/19」混在
- 直近は相対時刻、古いものは絶対日付。**判断ロジック明示** or 統一

## 結論

makxas-front は **MVP として機能要件はほぼ満たしている**が、**情報密度と一貫性で UX 負債が蓄積**している。

優先度 A の 5項目を片付けるだけでも、新規スタッフの定着率と日常運用のストレスが大きく改善されると予想する。Phase A は工数 2〜3 日で着手可能。

次のアクション提案: **Phase A の 5項目から着手** するか、優先度を関さんと議論してから決定する。
